# Method 15 — 5 Whys (五个为什么)

**Source**: Sakichi Toyoda, 1930s, 丰田生产方式 (Toyota Production System)。https://en.wikipedia.org/wiki/Five_whys

**Originator**: Sakichi Toyoda (豊田佐吉, 1867-1930) → 丰田生产方式的核心工具之一

## What it is
A root cause analysis method. When something is wrong, ask "why" 5 times. Each answer becomes the next question. The 5th-level answer is usually the systemic root cause, not the symptom.

## Why 5?
Not magic — depth required to get past proximate causes. 3 is often too few (you stop at blame). 7+ often loops or goes philosophical. 5 is the sweet spot for most engineering/process problems.

## The move
1. State the problem
2. Why? [answer 1]
3. Why? [answer 2]
4. Why? [answer 3]
5. Why? [answer 4]
6. Why? [answer 5 — usually the real root]
7. Design fix at the 5th level, not the 1st

## How to use this method in creative-ideation
1. User is stuck: "我不知道为什么 X 总是失败 / 不工作 / 没人在意"
2. Run 5 Whys on it
3. The 5th answer often *reframes* the problem — suddenly the solution is at a different level
4. Especially useful when user keeps applying surface-level fixes

## Best for
- 反复失败的流程
- 用户行为不符预期
- 工程/产品 bug
- "我一直这样做了还是不行"

## When NOT to use
- 创意生成 (这是诊断工具)
- 多个独立问题 (会纠缠)
- 没有具体失败事件

## Famous application
Toyota: "Machine stopped" → Why? Fuse blew → Why? Overload → Why? Bearing not lubricated → Why? Pump not pushing enough → Why? Pump shaft worn → Fix: replace shaft, not just the fuse.

## Output shape
- **Problem**: [what's wrong]
- **Why 1**: [proximate] → **Why 2**: → **Why 3**: → **Why 4**: → **Why 5 (root)**: [the systemic truth]
- **Fix at root level**: [the design move]
