# Method 02 — TRIZ (发明问题解决理论)

**Source**: Genrich Altshuller, 苏联海军专利审查员，1946-1985 年分析 200,000+ 专利后总结。https://en.wikipedia.org/wiki/TRIZ

**Originator**: Genrich Altshuller (苏联, 1926-1998)

## What it is
A systematic method for resolving technical contradictions. When improving parameter A degrades parameter B, TRIZ's 40 inventive principles show how other engineers historically resolved the same contradiction.

## How to use this method in creative-ideation
1. User describes a contradiction: "I want X but X causes Y"
2. Identify which 1-3 principles are likely to apply
3. Walk through how each principle would resolve the contradiction
4. Combine 2-3 principles if a single one doesn't fully resolve

## Best for
- Hardware/mechanical problems
- Software architecture tradeoffs (latency vs throughput, memory vs compute)
- Process design (speed vs quality, automation vs flexibility)
- Product design (cost vs durability, simplicity vs features)

## When NOT to use
- Pure aesthetic or emotional problems (use Defamiliarization)
- Open-ended exploration (use SCAMPER)
- Single-decision choices (use Pre-mortem)

## Output shape
- **Contradiction**: [A improves ↔ B degrades]
- **Principle(s)**: [#23 Feedback + #15 Dynamics]
- **Resolution sketch**: [how the principle(s) would resolve it in this domain]
