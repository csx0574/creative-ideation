# Method 17 — Mashup / Combinatorial Creativity (组合式创造)

**Source**: 概念可追溯至 Koestler (1964, *The Act of Creation*) 和 Boden (1990, *The Creative Mind*); 现代应用在 remix culture (音乐/电影/软件)。

**Originator**: Arthur Koestler (匈牙利-英国作家) + Margaret Boden (认知科学家)

## What it is
Novelty is rarely true novelty — it's recombination. The "two ideas that have never been in the same room" move. Most creative work is combinatorial: take A from one domain, take B from another, find where they overlap or how they can be fused.

## The move
1. List 2-3 unrelated domains
2. For each pair, ask: "What's structurally similar between them?"
3. Force the structural similarity to become a *product*: a hybrid, a metaphor, a tool, a name

## How to use this method in creative-ideation
1. User has 1 idea that's "fine but flat"
2. Pick 2 domains that are *unrelated* to the user's domain
3. For each domain, extract a *mechanism* (not the form)
4. Apply that mechanism back to the user's idea
5. Result: 1 idea that has user's intent + unexpected other-domain DNA

## Best for
- 用户说"想要新的"
- 已经用过 1-2 个方法，需要换工具
- 命名/品牌/Marketing
- 写"出圈"的内容

## When NOT to use
- 严肃技术问题 (用 TRIZ)
- 时间紧迫
- 用户没准备好接受跨域嫁接

## Famous examples
- **Game design**: "Pokemon = insect collection + monster battle + trading cards"
- **Music**: "DJ Shadow = hip-hop + sample-based archaeology"
- **Product**: "Uber = dispatch system + black car + app"
- **Book**: "House of Leaves = academic paper + horror + unreliable narrator"

## Output shape
- **Your idea**: [the flat one]
- **Unrelated domain A**: [the donor] → **Mechanism extracted**: [not the form]
- **Unrelated domain B**: [the donor] → **Mechanism extracted**:
- **Mashup**: [user's idea + A's mechanism + B's mechanism = new thing]
