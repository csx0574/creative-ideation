# Method 11 — Pattern Language (模式语言)

**Source**: Christopher Alexander, 1977, *A Pattern Language: Towns, Buildings, Construction*。https://en.wikipedia.org/wiki/A_Pattern_Language

**Originator**: Christopher Alexander (1936-2022, 建筑师, 加州大学伯克利分校)

## What it is
A structured vocabulary of 253 interconnected patterns that describe how to build humane environments — from "light on two sides of every room" to "alcoves" to "small public squares". Each pattern is a problem + a proven solution + the *forces* that make the solution work, and is linked to larger and smaller patterns.

## Why it matters for ideation
The framework generalizes beyond architecture: any complex domain can be analyzed as a system of named, linked patterns. Software (Gang of Four Design Patterns, 1994) explicitly borrowed from Alexander.

## How to use this method in creative-ideation
1. Identify the *forces* at play (the conflicting needs)
2. Search for an existing pattern that resolves them
3. If the pattern doesn't exist, *name* the new pattern: give it a clean concept name, describe the forces, describe the resolution, link it to neighbors
4. The act of naming makes the solution portable and teachable

## Best for
- 系统设计 (软件/产品/社区)
- 模式抽取 (从已成功项目中提炼)
- 解释"为什么这事之前没人讲清楚"
- 长尾复杂问题 (多个互相牵制的力)

## When NOT to use
- 短平快决策 (overkill)
- 单一变量问题 (用 TRIZ)
- 用户需要"一个点子"而不是"一个系统"

## Famous extensions
- "Gang of Four" Design Patterns (Gamma et al., 1994)
- Pedagogical Patterns (teaching patterns)
- Organizational Patterns (software team patterns)
- "Liberating Voices" oral tradition patterns
- Tim Ferriss / Tiago Forte PARA, etc. (informal pattern collection)

## Output shape
- **Name of pattern**: [the clean concept name, like "Window Place" or "Affinity Group"]
- **Forces**: [the conflicting needs this pattern resolves]
- **Resolution**: [the concrete design move]
- **Resulting context**: [what world this pattern creates]
- **Linked patterns**: [the bigger and smaller patterns this sits between]
