# Method 22 — "Everything Must Go" (清空-重启)

**Source**: Rob Walker & Joshua Glenn, 2009, *Significant Objects* 项目; 概念相关于 Subtract a Layer (创意约束, 来自上面 prompt 库)。

**Originator**: Rob Walker (记者) + Joshua Glenn (文化批评家) — 2009 年他们从 thrift stores 买 100 件垃圾, 然后请作家编故事, 重新标价, 验证 "narrative 增加 value" 的实验

## What it is
Take the work-in-progress, set it aside, then start from *wanting nothing*. "If I were to throw all of this away, what would I make *now*, knowing what I know?" Often the second pass is simpler, sharper, and better.

## The move
1. Acknowledge the existing work
2. Mentally (or physically) put it in a drawer
3. Imagine you're starting fresh today, but you have the *learning* from the discarded work
4. Build the v2 from scratch — don't reference the v1 except as a "what didn't work" log
5. Compare v1 and v2; usually v2 is 30-50% smaller, more focused, more honest

## How to use this method in creative-ideation
1. User has a draft/scaffold/spec that's grown to feel heavy
2. Frame: "Imagine v1 doesn't exist. You have the lessons. What do you make *now*?"
3. The 5-min v2 sketch often reveals the v1's real 10%
4. Then merge: take the v2's spine, transplant the v1's best 10%

## Best for
- 项目"长残了"
- 用户说"我之前走错路了"
- 文章/代码 refactor
- 设计大改

## When NOT to use
- 用户还有 momentum 和 v1 是好的 (premature)
- 时间紧
- 全新项目 (没东西可以"扔掉")

## Famous application
*Significant Objects* 项目: 100 件 thrift store 物件, 总买入价约 $128.74; 配故事后, eBay 售价约 $3,000+. 关键发现: 不是物件值钱, 是 narrative 让物件值钱 — 同时: 扔掉"值钱的"物件的 context 后, 重新写的 narrative 反而更有效.

## Output shape
- **Existing work being set aside**: [the v1]
- **Lessons from v1**: [what was learned, condensed to 3-5 bullets]
- **v2 (5-min sketch)**: [the new version, smaller and sharper]
- **What v1 had that v2 doesn't**: [often the bloated parts]
- **What v2 has that v1 didn't**: [often the honest core]
- **Merged direction**: [v2 spine + v1's best 10%]
