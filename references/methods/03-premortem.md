# Method 03 — Pre-mortem (事前验尸法)

**Source**: Gary Klein, 心理学家，原发表在 Harvard Business Review 2007 ("Performing a Project Premortem")。https://hbr.org/2007/09/performing-a-project-premortem

**Originator**: Gary Klein (美国认知心理学家, Naturalistic Decision Making 学派创始人)

## What it is
A decision tool that *assumes* the project has already failed catastrophically, then works backwards to identify why. The psychological trick: normal planning is biased toward success because people self-censor negative thoughts ("prospective hindsight" bias). Pre-mortem gives permission to imagine failure explicitly.

## How to use this method in creative-ideation
1. User is about to commit to a direction (writing a book, launching a product, picking a stack)
2. Say: "It's 12 months from now. This decision was a disaster. What went wrong?"
3. Generate 5-7 failure narratives
4. For each, ask: "Is this preventable? How?"
5. If 3+ failure modes are non-preventable → the decision is too risky; pick differently

## Best for
- Final-stage choices between 2-3 candidates
- "Should I commit to X for the next N months" decisions
- Pre-launch sanity check
- High-stakes commitments (career, money, time)

## When NOT to use
- Early exploration (use SCAMPER or Lateral Provocations)
- User is already in execution (use Oblique Strategies to break pattern)
- Cosmetic decisions (overkill)

## Famous variation
**Prospective hindsight** (Mitchell, 1989): people who *imagine* an event are 2-3x better at predicting reasons it might happen than people who *predict* it. Pre-mortem exploits this.

## Output shape
- **Decision under consideration**: [what they're choosing]
- **Failure mode 1**: [narrative] → **Preventable by**: [action]
- **Failure mode 2**: ...
- **Verdict**: proceed / modify / abandon
