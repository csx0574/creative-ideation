# Method 08 — First Principles (第一性原理)

**Source**: Aristotle (古希腊, 公元前 384-322), 经 Elon Musk 推广到现代工程语境。

**Originator**: Aristotle (概念) → 现代应用: Elon Musk (SpaceX, Tesla), Ray Dalio

## What it is
Reasoning from the most fundamental truths of a domain upward, instead of reasoning by analogy from existing solutions. "What is *definitely* true here? What follows from those truths alone?"

## The move
1. State the problem
2. List the *current* solutions and their assumptions
3. Break each assumption: which is actually necessary? Which is just inherited?
4. Strip to the foundational physical/economic/logical constraints
5. Reconstruct the solution from foundations

## How to use this method in creative-ideation
1. User says "X 太贵 / 太慢 / 太复杂" or "想重新思考 Y"
2. Push past the analogy-based answer: "为什么 X 必须存在？X 的每个组件是因为物理需要还是因为历史惯性？"
3. Build up from physics/economics/logic
4. Result: often an unconventional solution that's 5-10x cheaper/faster, OR a clear "no, the constraint is real" verdict

## Best for
- 重新思考成本/性能瓶颈
- 砍掉 inherited 复杂度
- 创业/产品定位 (vs 现有竞品)
- 解释"为什么这事没人做" — usually the answer is "因为没人用第一性原理想过"

## When NOT to use
- 灵感/brainstorm (不是结构化方法)
- 用户还没确定问题 (用 SCAMPER 之类)
- 纯美学问题 (用 Defamiliarization)

## Famous application
Musk on SpaceX 火箭: "火箭成本里 2% 是材料，剩下是...?" → 拆分后发现 70% 是 制造业 人工和 overhead → SpaceX 自己造 → 成本降到 1/3

## Output shape
- **Problem**: [the issue]
- **Current solution and its assumptions**: [the chain]
- **Foundational constraints only**: [what physics/economics actually requires]
- **Reconstructed solution**: [built up from foundations]
