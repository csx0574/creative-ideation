# Method 06 — SCAMPER

**Source**: Bob Eberle, 1971, 整理自 Alex Osborn 的 checklists (1963)。https://en.wikipedia.org/wiki/SCAMPER

**Originator**: Alex Osborn (BBDO 广告公司, 头脑风暴概念创造者) → 整理者 Bob Eberle

## What it is
A checklist of 7 mutation operators you apply to an existing thing to generate variants. It is *not* brainstorming-from-zero — it transforms something that already exists.

## The 7 operators
- **S**ubstitute: 替换某个组件/材料/人/步骤
- **C**ombine: 合并两个东西/目的/属性
- **A**dapt: 调整以适配新环境/新用户
- **M**odify / Magnify / Minify: 改变大小/频率/强度
- **P**ut to other use: 换个用途
- **E**liminate: 删掉某个组件/步骤
- **R**earrange / Reverse: 重新排序/倒序/翻转

## How to use this method in creative-ideation
1. User has an existing thing: 现有的 App / 文章 / 流程 / 工具
2. Apply each of the 7 operators as a prompt: "What if we substituted X with Y?"
3. Skip operators that clearly don't apply, but force at least 4
4. Best outputs come from combining 2-3 operators (e.g., Eliminate + Put to other use)

## Best for
- 扩张阶段 (expansion): user has 1 thing and wants 10
- Product iteration
- 现有文章/项目 卡壳
- Naming/positioning variants

## When NOT to use
- 空白阶段 (truly starting from nothing) — there's no existing thing to mutate
- Final decision between candidates (use Pre-mortem)
- Aesthetic/intrinsic quality problems (use Defamiliarization)

## Output shape
For each operator applied:
- **S/C/A/M/P/E/R applied to [existing thing]**: [the variant]
- **Promising?**: [Y/N + one-sentence why]
